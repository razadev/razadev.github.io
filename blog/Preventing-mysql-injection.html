Preventing mysql injection

Whenever I create web application that interact with mysqli database, I make sure I clean any data before entering into mysql.

There are 4 in-built function I normally use. You can read about them on PHP documentation website.

So the best way I use them, is to create a single function that return three of those functions.

By doing this, you completely reduce the chance of your database been injected.

Another secure (highly recommended) way of doing this is to combined PDO prepared statements and the clean_input function we created above.

You can use any of those methods, but from PHP.net they recommend you use PDO prepared statements, for extra security.

Please I will recommend you don’t used the old mysql function when connecting to a database. Unless you know how to play your way around this.
You should either use mysqli or Pdo.
